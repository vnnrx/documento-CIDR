1 — Classes IPv4 (históricas)

As classes A/B/C/D/E são históricas; atualmente usa-se CIDR — mas elas ajudam na leitura de documentos antigos.

Classe1º octeto (decimal)FaixaMáscara padrãoUso

E240–255240.0.0.0 – 255.255.255.255—Reservado/experimental
Observações
  • 127.0.0.0/8 é loopback.
  • 0.0.0.0 tem significado especial (rota default/esta rede).
  • Classes não determinam subnets modernas — use CIDR/VLSM.

3 — Faixas Especiais e Uso Prático

  • 100.64.0.0/10 — CGNAT (não roteável publicamente; para provedores).
  • 169.254.0.0/16 — Link-local (APIPA), autoconfiguração quando não há DHCP.
  • 127.0.0.0/8 — Loopback.
  • 224.0.0.0/4 — Multicast (classe D).
  • 192.0.2.0/24, 198.51.100.0/24, 203.0.113.0/24 — TEST-NETs para documentação.
  • 198.18.0.0/15 — Testes/benchmarks entre redes.

5 — Recomendações Práticas

  • LAN de usuários: usar /24 em 192.168.x.0/16 para facilidade. Segmentar por VLAN.
  • Gestão/OOB de provedores: usar 10.0.0.0/8 ou 172.16.0.0/12, planejar hierarquia por POP.
  • CGNAT: usar 100.64.0.0/10 (evita conflitos com 10/8/172/192 internos dos clientes).
  • Pontos-a-ponto: /31 quando suportado; economiza endereços.
  • VPNs: escolher blocos que não colidam com clientes; documentar.
  • Multicast: usar 239.0.0.0/8 para escopo administrativo quando necessário.

7 — Glossário Rápido

  • CIDR: Classless Inter-Domain Routing, ex.: 192.168.10.0/24.
  • VLSM: Subnetting com máscaras variadas.
  • Broadcast: endereço para todos os hosts do segmento.
  • Loopback: 127.0.0.1 (host local) ou /32 para roteadores.
  • CGNAT: NAT em escala de provedor — 100.64.0.0/10.
